# Security Policy

## Supported versions

This repository does not currently publish tagged releases. Security fixes are expected to land on the active default branch first.

| Version | Supported |
| --- | --- |
| Unreleased / default branch | Yes |
| Older snapshots and local builds | Best effort only |

## Reporting a vulnerability

Do not open a public issue for suspected vulnerabilities.

Preferred reporting paths:

1. use GitHub private vulnerability reporting for `14ag/VirtuaCam` if it is enabled
2. otherwise email `muriukipn@gmail.com`

Include:

1. affected component
2. reproduction steps
3. impact
4. logs, screenshots, or proof-of-concept details when safe to share
5. whether issue affects build flow, install flow, runtime behavior, or driver interaction

## Response policy

Current expected response:

- initial acknowledgement: best effort
- reproduction and triage: best effort
- public disclosure: after a fix or mitigation is available, when possible

Because repository does not currently publish versioned releases, fixes may first appear as commits on default branch before any packaged release process exists.
